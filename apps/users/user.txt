1️⃣ admin.py

Назначение:
Администрирование пользователей и управление их блокировкой через Django Admin.

Что реализовано:

Кастомные admin actions:

«Заблокировать пользователей»

Создаёт запись в UserBlock (если её ещё нет)

Устанавливает is_active = False

Показывает summary-уведомление в админке

«Разблокировать пользователей»

Удаляет записи UserBlock

Массово обновляет is_active = True

Кастомный UserAdmin:

Отображает ключевые поля пользователя (email, role, is_active, is_staff)

Поддерживает поиск и фильтрацию

Отдельная админка для UserBlock:

Просмотр причин и времени блокировки

Ключевая роль файла:
Админка является основным интерфейсом управления пользователями и точкой ручного контроля доступа.

2️⃣ apps.py

Назначение:
Конфигурация Django-приложения users.

Что делает:

Определяет имя приложения (apps.users)

Задаёт label="users" для корректной регистрации в Django

Роль:
Инфраструктурная часть, напрямую в логике не участвует.

3️⃣ jwt.py

Назначение:
Кастомизация логина через JWT с использованием email вместо username.

Что делает:

Наследуется от TokenObtainPairSerializer

Меняет поле аутентификации на email

Роль в системе:
Обеспечивает единый вход по email для всех пользователей API.

Критичный момент проверки:
Именно здесь определяется, кто может получить JWT-токен.

4️⃣ models.py
User

Назначение:
Кастомная модель пользователя.

Ключевые особенности:

Наследуется от AbstractUser

Email используется как:

поле логина (USERNAME_FIELD)

уникальный идентификатор

Введено поле role:

tenant

landlord

Роль:
Базовая сущность для всей системы (листинги, бронирования, отзывы, права доступа).

UserBlock

Назначение:
Хранение информации о блокировке пользователя.

Что хранит:

связь 1-к-1 с пользователем

причину блокировки

дату блокировки

Роль:
Аудит и контроль блокировок, отделённый от самой модели пользователя.

5️⃣ serializers.py
RegisterSerializer

Назначение:
Регистрация новых пользователей через API.

Поведение:

Принимает email, username, password, role

Корректно хеширует пароль через set_password

UserSerializer

Назначение:
Выдача информации о пользователях (в основном для админских API).

Особенности:

Все поля read-only

Используется для:

списка пользователей

просмотра статуса (is_active, is_staff, is_superuser)

6️⃣ urls.py

Назначение:
Маршрутизация API пользователей.

Основные эндпоинты:

/register/ — регистрация

/login/ — получение JWT

/refresh/ — обновление токена

/admin/users/ — список пользователей (только админы)

/users/<id>/block/ — блок / разблокировка пользователя

Роль:
Чёткое разделение:

публичных эндпоинтов

административных действий

7️⃣ views.py
RegisterView

Назначение:
Публичная регистрация пользователей.

EmailTokenObtainPairView

Назначение:
JWT-логин по email.

UserAdminViewSet / UserListAdminView

Назначение:
Просмотр пользователей через API.

Ограничения:

доступ только администраторам

UserBlockAPIView

Назначение:
Управление блокировкой пользователей через API.

Поведение:

POST → блокировка

DELETE → разблокировка

Роль:
Программный аналог admin actions, используется для админских панелей / фронта.

Итог по модулю users

Архитектура логичная и чистая

Блокировка реализована корректно по данным

Admin и API ведут себя одинаково

UserBlock отделён от User, что правильно для аудита

Soft Delete (мягкое удаление)
В проекте реализовано мягкое удаление через абстрактную модель SoftDeleteModel.
Удаление объектов не удаляет строку из БД, а помечает её как удалённую (is_deleted=True) и сохраняет время удаления (deleted_at).

objects — менеджер по умолчанию, возвращает только “живые” записи (is_deleted=False)

all_objects — возвращает все записи, включая удалённые

QuerySet поддерживает:

alive() — только живые

deleted() — только удалённые

restore() — массовое восстановление

hard_delete() — физическое удаление

Это позволяет безопасно скрывать удалённые данные из API/админки, но при необходимости восстанавливать или анализировать их.

Redis client
Для работы с Redis используется функция get_redis_client(), которая создаёт подключение по URL из переменной окружения REDIS_URL.
decode_responses=True включён для удобной работы со строковыми значениями и счётчиками (например, просмотры объявлений).

Модуль apps.listings отвечает за управление объявлениями жилья, их отображение в API и админке, а также за аналитику: просмотры, рейтинги, отзывы и историю поисков.

Используется связка:

Django + DRF — API

Redis — быстрый учёт просмотров

Celery + Celery Beat — асинхронные и периодические задачи

Soft Delete — безопасное удаление данных

1️⃣ Модели (models.py)
🔹 Listing

Основная модель объявления. Наследуется от SoftDeleteModel, что добавляет:

is_deleted

deleted_at

методы delete(), restore(), hard_delete()

менеджеры:

objects — только “живые”

all_objects — все (включая удалённые)

Основные поля:

owner — владелец объявления (AUTH_USER_MODEL)

title, description

адрес: country, city, postal_code, street, house_number, floor, apartment_number

price, currency

rooms

housing_type

parking_type

is_active

created_at, updated_at

Методы:

full_address() — собирает читаемый адрес для карты и API.

🔹 ListingViewStat

Накопительная статистика просмотров.

OneToOne с Listing (primary_key=True)

views_total

обновляется при flush из Redis

Используется для админки, сортировки и аналитики.

🔹 ListingViewEvent

История просмотров пользователями.

FK на Listing

FK на User

created_at

индексы по (user, created_at) и (listing, created_at)

Используется для endpoint’а “моя история просмотров”.

🔹 SearchQuery

История поисковых запросов.

query

user (nullable)

created_at

индексы для аналитики

Используется для endpoint’а “popular searches”.

2️⃣ Права доступа (permissions.py)
🔐 IsLandlord

Разрешает действие только авторизованному пользователю с ролью landlord.

🔐 IsOwnerOrReadOnly

чтение доступно всем

изменение / удаление — только владельцу объявления

3️⃣ Сериализация (serializers.py)
🔹 ListingSerializer

сериализует все поля Listing

добавляет вычисляемое поле full_address

avg_rating и reviews_count — только для чтения (из annotate)

поле owner — read-only

4️⃣ Админка (admin.py)
🔹 ListingAdmin

Админка расширена аналитикой и удобством:

Отображаемые поля:

цена с валютой

тип жилья, парковка, комнаты

активность

владелец

просмотры

средний рейтинг

количество отзывов

ссылка на Google Maps

soft-delete статус

Оптимизация:

select_related("owner")

аннотации:

_views_total (Subquery из ListingViewStat)

_avg_rating (Avg)

_reviews_count (Count)

Действия админки:

восстановление soft-deleted объявлений

hard delete

копирование объявления

🔹 ListingViewStatAdmin

Позволяет контролировать статистику просмотров напрямую из админки.

5️⃣ Сигналы (signals.py)
📩 Уведомления админа

При создании Listing:

отправляется письмо администраторам

🔍 Поиск дубликатов

при сохранении Listing ищутся дубликаты по всем ключевым полям

при обнаружении:

уведомление админу

если объявление новое → запускается Celery-задача на удаление через заданное время

Это защищает систему от спама и случайных дублей.

6️⃣ Celery + Redis (tasks.py)
🔹 Учёт просмотров (Redis)
listing:<id>:views

track_listing_view

увеличивает счётчик просмотров в Redis

просмотры владельца не учитываются

🔹 Flush просмотров в БД
flush_listing_views_to_db

периодически вызывается Celery Beat

читает ключи из Redis через SCAN

атомарно переносит значения в ListingViewStat.views_total

удаляет ключи из Redis

Позволяет:

не нагружать БД при каждом просмотре

хранить итоговую статистику надёжно

🔹 История просмотров
save_listing_view_event

сохраняет факт просмотра пользователем

используется для истории просмотров

🔹 Удаление дубликатов
delete_listing_if_still_duplicate

проверяет, остались ли дубликаты спустя время

если да — soft delete (или hard delete при отсутствии soft delete)

7️⃣ API (views.py)
🔹 ListingViewSet

Основной DRF ViewSet.

Функциональность:

CRUD объявлений

сортировка:

по дате

по рейтингу

по просмотрам

по количеству отзывов

подсчёт просмотров при retrieve

сохранение истории просмотров

Дополнительные endpoints:

POST /listings/{id}/copy/ — копирование объявления

GET /listings/popular/ — популярные объявления

GET /listings/my-view-history/ — история просмотров пользователя

🔹 listings_search

Функциональный endpoint поиска:

фильтрация по адресу, цене, валюте, комнатам, типу жилья

сортировка

пагинация

запись поисковых запросов

предупреждения при некорректных фильтрах (например price без currency)

🔹 popular_searches

Возвращает самые популярные поисковые запросы пользователей.

8️⃣ URLs (urls.py)

/listings/ — CRUD + ViewSet

/listings-search/ — поиск

/listings/search/popular/ — популярные запросы

9️⃣ Архитектурные решения

✔ Redis используется только как временное хранилище счётчиков
✔ База данных — источник истины
✔ Celery разгружает API
✔ Soft delete предотвращает потерю данных
✔ Админка используется как аналитический инструмент
